AAB RELEASE BUILD INSTRUCTIONS

Goal:
Google Play normally expects Android App Bundle (.aab), not only APK.

GitHub Actions command usually needed:
./gradlew bundleRelease

Expected output:
app/build/outputs/bundle/release/app-release.aab

Release signing:
Use the same keystore/signing config that is currently used for update builds.
Do not change applicationId.
Always increase versionCode before uploading a new Play release.

Current app identity should remain:
applicationId: com.alensgarage.worldcupfan2026

Checklist before Play upload:
1. Build release AAB.
2. Download artifact from GitHub Actions.
3. Upload AAB to Internal testing first.
4. Install from Google Play internal test link.
5. Test language, results, dark mode, My Team, Share Poster, Host Cities and Players.
